\# 钢琴网站产品需求文档（PRD）



\## 1. 项目概述  

\### 1.1 背景与目标  

开发一款基于 Vue 框架的移动端钢琴网站，支持 88 键全键盘演奏，实现真实的钢琴音色响应，满足音乐爱好者随时随地在手机上进行弹奏、练习及移调练习的需求。  



\### 1.2 目标用户  

\- 钢琴初学者，希望利用碎片时间熟悉键盘位置。  

\- 音乐爱好者，需要便携的移调工具进行即兴演奏。  

\- 专业音乐人，用于快速试听和演示旋律。  



\### 1.3 核心价值  

\- \*\*全平台适配\*\*：手机横竖屏均能正常使用，尤其注重横屏下的完整键盘体验。  

\- \*\*真实演奏感\*\*：低延迟的触摸反馈与高质量钢琴音色。  

\- \*\*移调便捷\*\*：一键上下移调，覆盖 ±12 个半音范围。  

\- \*\*可视导航\*\*：底部缩略图辅助快速定位当前演奏区域。

\## 2. 功能需求  



| 模块 | 功能点 | 描述 |

|------|--------|------|

| 钢琴键盘 | 88 键完整展示 | 包括白键 52 个、黑键 36 个，视觉区分明确，支持触摸发声。 |

| 移调功能 | 上下移调 | 提供“+”、“-”按钮，每次操作移调 1 个半音，范围 -12 \~ +12。移调后所有琴键的音高整体偏移。 |

| 底部缩略图 | 横向滚动导航 | 显示 88 个键的缩小概览，当前视口区域高亮显示，支持拖拽或点击跳转。 |

| 屏幕方向适配 | 横屏 / 竖屏 | 默认横屏；竖屏时键盘整体缩小并允许横向滚动，保证所有键可见。 |

| 声音引擎 | 触摸发声 | 点击或触摸琴键区域即发出对应音高的钢琴声音，支持多点触控。 |

| 视觉反馈 | 按键高亮 | 触摸琴键时，对应按键有按下态视觉反馈（颜色变化或阴影）。 |



\### 2.1 键盘布局与交互  

\- 白键与黑键按标准钢琴键盘排列（C1 \~ B8）。  

\- 黑键高度约为白键的 60%，宽度适当收窄，在手机上保持可点区域不小于 32×32 dp。  

\- 支持手指滑动连续演奏（即手指在键盘上滑动时，依次触发经过的琴键声音）。  



\### 2.2 移调逻辑  

\- 移调数值显示在界面上方（如“移调：+3”）。  

\- 移调改变的是实际发出的音频频率，键盘视觉上的音名标签可随移调同步更新（可选，建议实现）。  

\- 移调后，88 个键的触发表面对应新的音高，例如移调 +1，C 键发出的声音为 C#。  



\### 2.3 底部缩略图  

\- 缩略图高度约 40\~50px，宽度与键盘总宽度成比例。  

\- 用矩形块表示每个键（白键浅色、黑键深色）。  

\- 当前屏幕可见的键盘区域在缩略图中用边框或高亮蒙层标示。  

\- 支持左右拖动缩略图，主键盘区域同步滚动到对应位置。

| 类型 | 要求 |

|------|------|

| 横竖屏适配 | 监听屏幕方向变化，动态调整键盘布局和尺寸，所有元素重绘。 |

\## 4. 技术方案建议  



\### 4.1 技术栈  

\- \*\*框架\*\*：Vue 3（或 Vue 2 + Composition API）  

\- \*\*音源\*\*：Web Audio API（推荐）或 HTML5 Audio 元素。使用预加载的钢琴单音样本（如 .mp3 或 .ogg），或使用采样器动态生成波形。  

\- \*\*触摸事件\*\*：`touchstart`、`touchmove`、`touchend`，需阻止页面滚动（`preventDefault`）。  

\- \*\*布局\*\*：CSS Flex + Grid，横屏时键盘宽度为 100vw，高度自适应；竖屏时设置键盘容器横向滚动（`overflow-x: auto`）。  

\- \*\*状态管理\*\*：Vue 响应式数据管理当前移调值、缩略图滚动位置等。  



\### 4.2 钢琴键盘实现  

\- \*\*DOM 方式\*\*：动态生成 88 个键的 `<div>`，通过 CSS 定位排列，便于绑定事件和视觉反馈。  

\- \*\*Canvas 方式\*\*：统一绘制，性能更好，但触摸坐标映射相对复杂。建议优先采用 DOM 元素，开发维护简单。  



\### 4.3 音频处理  

\- 使用 `AudioContext`，预先加载 12 个半音样本（因为移调可通过改变播放速率实现，或直接加载全部 88 个样本但体积较大）。  

\- 推荐方案：加载一个八度的 12 个基础样本，移调时通过修改 `playbackRate` 实现，大幅减少加载资源。  

\- 移动端需注意 `AudioContext` 必须在用户首次触摸时 `resume`（浏览器限制自动播放策略）。  



\### 4.4 缩略图组件  

\- 独立于主键盘的横向滚动容器，监听 `scroll` 事件并与主键盘滚动同步。  

\- 主键盘滚动时，缩略图高亮区域自动更新；反之亦然。

\## 5. 用户界面与交互细节  



\### 5.1 布局（横屏为主）  

```

┌─────────────────────────────────┐

│      移调：-2  移调：+2         │  顶部工具栏

├─────────────────────────────────┤

│                                 │

│         钢琴键盘区域             │  （横向滚动或全宽）

│        （88 键自适应排列）       │

│                                 │

├─────────────────────────────────┤

│        底部缩略图（可拖动）       │

└─────────────────────────────────┘

```



\### 5.2 触摸与反馈  

\- 按下白键：浅灰色背景；黑键：深灰色背景。  

\- 手指在键盘上滑动：`touchmove` 事件计算当前触摸点所在的琴键，触发发声。  

\- 多点触控：支持多个手指同时按下不同琴键（同时发声）。  



\### 5.3 移调交互  

\- 显示当前移调值，点击“+1”或“-1”即时生效。  

\- 移调值范围 -12 \~ +12，超出时按钮置灰或提示。  



\### 5.4 缩略图交互  

\- 支持手指左右滑动缩略图区域，主键盘跟随滚动。  

\- 点击缩略图上的任意位置，主键盘直接跳转到对应位置。

\## 6. 适配与兼容性策略  



\### 6.1 横屏与竖屏  

\- 使用 CSS 媒体查询检测方向，设置不同样式。  

\- 横屏时，键盘容器宽高比适合 88 键完整显示（可能需要横向滚动）。  

\- 竖屏时，键盘容器高度缩小，强制横向滚动，确保所有键可触及。  



\### 6.2 移动端优化  

\- 设置 `viewport`，禁止用户缩放。  

\- 触摸按钮、缩略图区域需有足够大点击区域（44×44 dp）。  

\- 使用 `-webkit-tap-highlight-color` 移除默认点击高亮。  



\### 6.3 声音兼容  

\- iOS 上需要用户首次点击页面任意位置来启动 `AudioContext`。  

\- Android 上同样遵循自动播放策略，可在页面加载时显示“点击屏幕激活声音”提示层。

\## 9. 风险与应对  



\- \*\*音频自动播放限制\*\*：增加启动引导，首次触摸时初始化 `AudioContext`。  

\- \*\*性能问题（大量 DOM 元素）\*\*：使用虚拟滚动或 Canvas 渲染，但优先级先采用 DOM，若性能不达标则重构为 Canvas。  

\- \*\*黑键触摸区域过小\*\*：适当扩大热区，通过 `::after` 伪元素扩大点击区域。  

\- \*\*横竖屏切换导致键盘重绘卡顿\*\*：使用 `resize` 事件节流，更新布局时仅重新计算样式，避免重建 DOM。
